# Sample Local Structure

This is a fictional example showing how a user might organise their private `local/` folder.

Do not use real personal data in examples.

## Example folder

```text
local/
  README.md
  profile.md
  idea-inbox.md
  project-index.md
  decision-log.md
  active-projects.md
  model-preferences.md
```

## Example file purposes

| File | Purpose |
|---|---|
| `local/README.md` | Explains the user's private local folder. |
| `local/profile.md` | Stores private user preferences and context. |
| `local/idea-inbox.md` | Stores messy thoughts before they are processed. |
| `local/project-index.md` | Tracks active, paused, archived, and future projects. |
| `local/decision-log.md` | Stores saved decisions from the Save Decision workflow. |
| `local/active-projects.md` | Gives a quick view of current priorities. |
| `local/model-preferences.md` | Stores local preferences for cloud, local, or hybrid model routes. |

## Fictional example

```text
local/
  README.md
  profile.md
  idea-inbox.md
  project-index.md
  decision-log.md
```

A fictional user might use this structure as follows:

- `profile.md` contains their preferred working style.
- `idea-inbox.md` contains rough ideas that have not been routed yet.
- `project-index.md` tracks which ideas became real projects.
- `decision-log.md` records what the system decided to save.
- `README.md` explains how their local folder is organised.

## Safety reminder

This example is safe to commit because it contains no real private data.

Real local files should stay inside `local/` and should not be committed.

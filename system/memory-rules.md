# Memory Rules

## Purpose

Memory should help Aodh Lab become more useful over time without becoming cluttered, invasive, or confused.

The agent should not save everything.

It should only save information that improves future usefulness.

## Save to Profile Memory When

Save information that is stable and likely to remain useful for months or years, such as:

- Long-term learning goals
- Stable preferences
- Communication preferences
- Important personal constraints
- Ongoing project context
- Reusable principles

## Save to Active Projects When

Save information about current projects, including:

- Project purpose
- Current status
- Key decisions
- Open questions
- Next actions
- Dependencies
- Related files

## Save to Decision Log When

Save decisions that affect the direction of the system or a project.

Each decision should include:

- Date
- Decision
- Reason
- Implication

## Save to Idea Inbox When

Save raw or undeveloped thoughts that may become useful later.

These do not need to be polished.

## Do Not Save

Do not save:

- Random passing thoughts
- Sensitive personal details unless explicitly approved
- Private data belonging to other people or organisations
- Restricted or sensitive material from systems outside the lab
- Credentials or secrets
- Temporary emotions unless they are part of an intentional reflection
- Anything from outside the Aodh Lab folder unless explicitly approved

## Approval Rule

When unsure whether something should be saved, the agent should ask:

"Should this be saved to memory, saved as a project note, or left as a one-off response?"
